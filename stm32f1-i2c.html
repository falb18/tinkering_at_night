Notes about the setup and of the I2C on the STM32F103C8T6

Posted by Felipe Arturo López Bonilla on

In this post I try to setup and use the I2C peripheral on the STM32F103C8T5 also know as "bluepill". To be clear I'm not familiar with this kind of microcontrollers but I think these notes will be helpfull for someone else who also wants to do it's own implementation of I2C reading and writing. Besides, as with most 32-bit microcontrollers, there are more registers to setup in order to use the peripheral which sometimes can be confusing, specially when you are a novice like me.

I'm using the libopencm3 library to access the registers. Although this library already has functions to setup the registers correctly, I want to do most of the implementation setting or clearing the corresponding registers' bits. I choosed to do it this way since I believe it gives a better understanding what's going on under the hood so later you can have a better idea why the peripheral is not working.

For this project the microcontroller will communicate to a DS3231 chip, real-time controller, which will act as a slave device. The microcontroller has two I2C peripherals, in this case I'm using I2C1. The clock frequency of the micrcontroller is setup to 24 Mhz.

Setup the I2C on the STM32F1

First, we enable the clocks for the I2C pins, which are in PORTA; for the alternate function on those pins, and for the peripheral itself.

Second, setup the gpios as open-drain output with the alternate function activated. It's important that the pins are set as open-drain since a pull-up resistor is attached to those pins.

Finally, the following parameters are set to configure the I2C1:

  • The microcontroller will be in MASTER mode.
  • The I2C frequency is set to standard (100 Khz).
